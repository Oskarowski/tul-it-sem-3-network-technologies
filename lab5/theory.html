• DNS system:

    The Domain Name System (DNS) is a hierarchical decentralized naming system that translates human-readable domain names into numerical IP addresses. It is a fundamental component of the internet, serving as a directory for mapping domain names to their corresponding IP addresses.

  • List of Top-Level Domains:

    Top-Level Domains (TLDs) represent the highest level in the DNS hierarchy. The top-level domain names are installed in the root zone of the namespace. Some common TLDs include:

    TLD Description
    .com (commercial)
    .org (organization)
    .net (network)
    .gov (government)
    .edu (education)
    .us United States
    .uk United Kingdom
    .jp Japan
    .pl Poland

  • Hierarchical structure of DNS:

    The DNS has a hierarchical structure, consisting of multiple levels. At the top are the root servers, followed by top-level domain servers, authoritative domain servers, and the local DNS resolver.

    • The root domain:

      is at the apex of the domain name hierarchy. Above it is the top-level domains, further divided into second-level domains, third-level domains, and so on.

    • The top-level domains:

      include generic domains, such as .com, .net, and .org, and country code top-level domains, such as .uk and .us. The second-level domains are typically used to identify an organization or business. For example, the domain name google.com consists of the second-level domain Google and the top-level domain .com.

    • Third-level domains:

      identify a specific host or service associated with a domain name. For example, the domain name www.google.com consists of the third-level domain www, the second-level domain google, and the top-level domain .com.

    • Fifth-level domains:

      are typically used to identify a particular resource within a domain. An example of a fifth-level domain is docs.google.com, which is used to access Googleâ€™s online document storage service.

  • DNS Queries:

    A DNS query is a message that a client sends to the DNS server. It contains a list of questions that the DNS server answers. A DNS query can contain multiple questions that the server will reply to, but a server might also reply with its own additional information.

  • DNS Delegation:

    DNS delegation involves the distribution of authority for a domain across multiple servers, allowing for efficient management of domain namespaces.

  • DNS record:

    DNS delegation involves the distribution of authority for a domain across multiple servers, allowing for efficient management of domain namespaces.

  • DNS and reverse DNS:

    DNS: The process of translating a domain name into an IP address.

    Reverse DNS: The reverse of DNS process, mapping an IP address to a domain name.

  • Authoritative answer non authoritative:

    • Authoritative Answer:

      Provided by a DNS server that has the specific requested information. It is considered trustworthy and definitive.

    • Non-Authoritative Answer:

      Provided by a server that does not contain original source files of domain's zone.

  • DNS resolver, DNS cache, file: /etc/hosts:

    • DNS Resolver:

      Server that receives DNS queries from clients and finds the corresponding IP addresses. It can be local or provided by the ISP (Internet Service Provider)

    • DNS Cache:

      Stores previously resolved DNS queries, reducing the need to query authoritative servers repeatedly.

    • File: /etc/hosts:

      An operating system file that maps IP addresses to hostnames locally, bypassing DNS. It is commonly used for manual address resolution or to override DNS entries.




  • Protocols POP3, IMAP, SMTP:

    • POP3 (Post Office Protocol 3):

      Retrieves emails from a server to a client device. Emails are completely stored on the device after being downloaded from the mail server. As a result, an internet connection is only required for syncing the inbox. Typically deletes mail from the server after downloading.

    • IMAP (Internet Message Access Protocol):

      Allows multiple devices to access the same mailbox, keeping messages on the server and synchronizing changes across devices.

    • SMTP (Simple Mail Transfer Protocol):

      Connection oriented, mainly for managing the sending of emails between servers.

    Command Description
    HELO Provides the identification of the sender (i.e., the host name)
    MAIL Specifies the originator of the mail
    RCPT Specifies the recipient of the mail
    DATA Specifies the beginning of the mail
    QUIT Closes the TCP connection
    RSET Aborts the current mail transaction, but the TCP connection remains open
    VRFY Used to confirm or verify the username
    QUIT Commit message and close the channel

  • NNTP protocol:

    NNTP (Network News Transfer Protocol): Primarily used for reading and posting messages to UseNet (worldwide distributed discussion system) newsgroups, providing a distributed discussion system.

  • E-mail mailbox, parameters, aliases, address:

    • Email Mailbox:

      Storage location for received emails, accessible through email clients.

    • Parameters:
      • Storage limit
      • Messages per day
      • Advanced spam & virus protection
      • Recipients per message (To, Cc, and Bcc)
      • Message limit
      • Alias limit
      • Footer template
    • Aliases:

      Alternate email addresses (forwarding address) that redirect messages to the primary mailbox. With one e-mail account, client can have multiple e-mail addresses and one mailbox.

    • Address:

      Unique identifier for e-mail mailboxes, typically in the format local-part@domain

  • E-mail structure: header, separator:

    • Header:

      Contains metadata about the email like sender, recipient, subject, date.

    • Separator:

      Marks the boundary between one message and another in an email conversation thread.

  • Attachments, e-mail coding, MIME:

    • Attachments:

      Additional files included with an email.

    • E-mail coding:

      The source code of an email will contain the header information for the email and will often include HTML code to style the rest of the message.

    • MIME (Multipurpose Internet Mail Extensions):

      An email standard that extends email capabilities to support various content types, attachments.

  • Mailing lists, e-mail gate:

    • Mailing Lists:

      Group of email addresses used for sending messages to multiple recipients simultaneously.

    • Email Gate:

      Acts as a gateway, managing the flow of emails, protecting server, client from malicious mails, spams.

  • E-mail security, SSL, TLS, PGP:

    E-mail security - describing different procedures and techniques for protecting email accounts, content.

    • SSL & TLS:

      SSL (Secure Sockets Layer) and TLS (Transport Layer Security): Encryption protocols ensuring secure communication between email clients and servers.

    • PGP (Pretty Good Privacy):

      Encryption method for securing email content and providing digital signatures for verification.

  • Spam, blacklists, graylists, spam filtering :

    E-mail security - describing different procedures and techniques for protecting email accounts, content.

    • Spam:

      Unwanted communique

    • Blacklists:

      Collection of entities from which we are not interested in communications.

    • Graylists:

      Serve as a repository for entities that have not yet been classified into either a blacklist or whitelist.

    • Spam Filtering:

      Techniques and tools to identify and block spam messages, including content analysis and blacklisting, virus scans.
